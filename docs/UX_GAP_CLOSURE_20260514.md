## 一、审计方法

全链路追踪了"数据产生 → 数据传递 → 界面渲染"的完整路径：

```
Pipeline._runtime_data() → progress_callback → JobStore → SSE/Polling → Web Frontend → DOM
```

对比了后端已有的 25 项评分指标、8 种 YAML 假设、16 个数据集、3 种生成模式、6 种变异算子——与实际前端暴露的内容之间的差距。

## 三、已实施修复

### 修复 1：评分卡片可视化（P0 #1, #6）

替换了 `renderCandidateDetail()` 中的 raw JSON 文本块，新增 `renderScorecardVisual()`：

- **决策带徽章**：彩色圆角标签（绿=可提交 / 黄=需优化 / 蓝=待研究 / 红=废弃 / 粉=硬门禁阻断）
- **总分进度条**：颜色随分数变化（≥85 绿 / ≥70 黄 / ≥50 蓝 / <50 红）
- **先验 8 维度**：每维度一行，带彩色条 + 图标 + 实际分 + 权重百分比
- **实证 16 项**：表格式展示，通过/失败图标 + 实际值 + 得分 + 硬门禁标记
- **提交清单 7 项**：逐项状态指示
- **排序分公式分解**：蓝色信息框显示 `检查分 + 检查奖励 + 指标加成 − 云端惩罚` 完整演算

### 修复 2：生命周期时间线（P0 #2）

新增 `renderLifecycleSteps()`，用 6 步彩色标签 + 箭头连接展示 Alpha 旅程：

```
生成 → 本地评分 → 表达式验证 → 官方回测 → 综合评分 → 门禁判定
```

每步用不同颜色状态：绿色=已完成 / 蓝色=进行中 / 灰色=待完成 / 红色=失败

### 修复 3：生成溯源标签（P0 #3, #7, #8）

新增 `renderProvenance()`，在详情页顶部用彩色徽章显示：

- **生成模式**：紫色=假设驱动 / 黄色=经验反馈 / 灰色=随机探索
- **家族分类**：蓝色标签
- **数据集 ID**：绿色标签
- **假说摘要**：单行省略显示

### 修复 4：AlphaCheck 结果可视化（P0 #4）

新增 `renderCheckResults()`，用结构化方式展示 BRAIN 检查结果：

- 错误/警告/通过 计数摘要（红/黄/绿）
- 每项检查一行，带通过/失败图标 + 严重级别标签 + 详情

### 修复 5：数据集与校准状态（P0 #5, P1 #8）

Pipeline `_runtime_data()` 新增字段：
- `active_dataset_id`：当前使用数据集
- `scoring_calibrated`：评分参数是否已校准

Web 监控面板新增第 9 个瓦片「数据集」，显示当前数据集和校准状态。

### 修复 6：云 Alpha / 回测槽详情增强

`renderCloudDetail()` 和 `renderBacktestDetail()` 也获得了结构化评分视图（合成评分卡片），不再仅显示 raw JSON。
