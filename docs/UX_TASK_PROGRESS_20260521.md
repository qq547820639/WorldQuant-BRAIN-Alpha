## 本轮 UX 落差与修复方案

| UX 落差 | 前端修复 |
| --- | --- |
| 核心流程入口分散，用户需要在侧栏、洞察卡和操作条之间来回寻找下一步。 | 在 `index_template.html` 增加 `workflowNav`，由 `app.js/renderWorkflowNav()` 动态渲染生产、回测、检查、提交、诊断 5 步入口。 |
| 侧栏配置密度高，关键操作与高级参数混在一起。 | 使用 `control-section` 分组，将连接、生产、市场参数、云端同步和高级策略拆开，生产按钮固定在首屏主分组。 |
| 诊断类卡片与生产链路并列展示，干扰候选、达标和提交判断。 | `renderInsight()` 保留生产流程为主卡片，将辅助追踪与诊断改成可折叠分组，仅在诊断视图自动展开。 |
| 空列表只有一句文字，用户不知道下一步该做什么。 | 新增 `emptyStateHtml()`、`emptyStateHint()` 和 `emptyStateActions()`，表格与移动卡片共用带 CTA 的空状态。 |
| 按钮、流程卡、筛选与输入反馈不一致。 | 补齐全局 `button:focus-visible`、`workflow-step`、`view-guidance`、`filter-chip` 等统一 hover/focus/active 状态。 |
| 小屏下顶部信息和工具条易拥挤。 | 更新 1380/1120/720px 断点，流程导航、洞察卡、工具条、控制入口和移动卡片改为自适应单列或三列。 |
| 缺少前端模块级测试，UX 调整容易破坏交互逻辑。 | 新增 `tests/test_web_frontend_modules.py`，为每个前端模块建立测试矩阵，并用 bundled Node 执行渲染/状态/交互契约测试。 |

## 前端模块测试覆盖

| 模块 | 覆盖重点 |
| --- | --- |
| `js/api-client.js` | API 路径、CSRF header、错误码到用户提示映射。 |
| `js/state.js` | 嵌套状态读写、merge、监听器、缓存、新鲜检查与可提交判断。 |
| `js/utils.js` | HTML/属性转义、阶段名称、检查名称、风险说明与状态路径渲染。 |
| `js/view-model.js` | 表达式归一化、候选/生命周期去重、运行态数组选择。 |
| `js/components/*.js` | Toast、Spinner、Modal、Progress、Table 的渲染、可访问角色、空状态和安全转义。 |
| `js/views/charts.js` | 离线 canvas 图表兜底、空数据提示、候选数据绘制。 |
| `js/views/detail.js` | 检查详情、建议提示、弹窗显示和动态文本转义。 |
| `js/views/monitor.js` | 监控瓦片与回测槽位渲染。 |
| `js/views/production.js` | 生产入口导出、冲突操作 guard、阻断反馈。 |
| `js/app.js` | 流程导航、空状态 CTA、忙碌态互斥、视图编排和响应式 UX 契约。 |
