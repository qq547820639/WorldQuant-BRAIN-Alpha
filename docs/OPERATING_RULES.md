# Research Operations Rules

本项目按 v2.1 作战手册重构，工程规则如下。

## 来源标记

- `官方`：平台文档、后台、系统实际返回。
- `经验`：社区或历史研究经验，不保证固定。
- `推测`：根据现象推断，不当作硬规则。
- `待验证`：需要在 Brain 后台确认。
- `协议`：以用户签署协议为准。

## 提交流程

1. 本地生成候选。
2. 本地 lint 与质量预筛。
3. 只把较好候选送回测前预检/官方模拟。
4. 读取官方 Alpha ID 与指标。
5. 结构化评分。
6. 生产门禁。
7. 账户安全门禁。
8. 官方 alpha check。
9. 提交。
10. 记录结果，继续生产。

## 运行配置

- 所有默认运行参数放在 `config/run_config.json`。
- `run_pipeline.py` 是编辑器直接运行入口；不需要写命令行参数。
- `launch_web.py` 是编辑器直接运行的本地控制台入口；端口和是否打开浏览器由配置文件控制。
- CLI 仍可临时覆盖配置，但配置文件是唯一默认来源。
- 凭据优先走环境变量；不要把账号、密码或 Token 写入配置文件后提交或分享。

## 官方 API 限流

- 出现 HTTP 429 时，系统会按 `Retry-After` 或配置的 backoff 等待并重试。
- 字段列表和算子列表会写入 `ops.official_api.cache_dir`，默认 24 小时内复用。
- 如果上下文接口被限流但本地已有缓存，允许使用旧缓存继续运行。
- 已经限流时应减少候选、验证、模拟预算，或暂停一段时间再跑。
- Web 控制台中的连续生产、云端同步、批量检查和提交互斥执行，避免重复点击或同时争用官方接口。
- 默认要求提交前存在未超过 24 小时的云端缓存；缓存缺失或过期时先同步云端数据。

## 账户安全

自动提交必须同时满足：

- 官方模拟指标存在。
- Sharpe、Fitness、Turnover、Returns、Drawdown、Sub-universe、Correlation、Concentration 达到配置阈值。
- 官方 pass/fail 通过。
- 不是重复表达式或微小变体。
- 没超过每日和单次运行自动提交上限。
- 与上次自动提交间隔足够。

## 放弃标准

候选出现以下情况应进入归档或重构：

- 本地质量低。
- 语法预检或官方模拟失败。
- 官方指标明显不达标。
- 相关性高。
- 表达式只是已有逻辑的微小参数变化。
- 经济逻辑无法解释。
