## 修复 check_batch 链路缺少 fields/operators 同步

**问题**：用户确认自动/手动同步已覆盖 fields/operators，但 `check_batch` 链路只刷新云端 Alpha，未顺带刷新 fields/operators 缓存。

**根因**：`brain_alpha_ops/web.py` 的 `refresh_cloud_context_for_check()` 函数（第 3957 行）只调用了 `api.list_user_alphas()`，没有像 `run_sync_job` 那样接着调用 `api.list_fields()` 和 `api.list_operators()`。

**修复**：在 `refresh_cloud_context_for_check` 中，`list_user_alphas` 成功返回后，补上：
```python
try:
    region = api._market_scope.get("region", "USA")
    api.list_fields("all", region)
    api.list_operators("all")
except Exception:
    pass
```

**修改文件**：`D:\Works\WorldQuant BRAIN Alpha\brain_alpha_ops\web.py`（第 3983-3988 行）

**验证**：手动审查代码逻辑正确，fields/operators 调用由独立 try/except 包裹，失败不影响主流程。
